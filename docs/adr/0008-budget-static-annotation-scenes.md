# Budget static-first Annotation scenes without weakening semantics

V1 treats a static blog as the primary delivery path: an ordinary Annotation scene requires no client-only runtime, while hydrated React remains an optional host choice. We accept the closed Scene plan validator and finalizer shipped by #16 as the semantic baseline, enforce fixed ESM, consumer-bundle, CSS, font, transport, SSR, and npm-package byte budgets immediately, retain source maps and one default self-hosted font, and require every additional Visual style or font to be explicit opt-in. Timing limits begin as five successful `main` calibration reports before becoming blocking because shared runners are noisy; byte limits are deterministic and block immediately. This deliberately rejects shrinking bundles by weakening validation, discarding plan information, moving compilation into React, or making browser runtime mandatory.
