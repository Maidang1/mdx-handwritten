# Keep Scene compilation framework-neutral and render through adapters

MDX Handwritten keeps `mdx-handwritten-scene` as the leaf, framework-neutral Module that materializes a closed Scene plan through `createScenePlan`; remark syntax, React, DOM serialization, sanitization policy, CSS, browser layout, storage, and AI remain outside its Interface. `remark-mdx-handwritten` owns author-syntax validation and build-output policy, while its element output and `mdx-handwritten-react` are separate Annotation renderers over the same plan and `mdx-handwritten-theme` is optional Visual style over their stable semantic DOM. The fixed eight Annotation gesture directives and `handwrittenComponents` map never absorb `hw-scene` or `HandScene`.

For the V1 component path, remark materializes the plan once and passes it to `HandScene`, so Semantic corrections and reviewed plans cannot be lost through runtime re-derivation. Direct React usage also retains the existing `recipe + source + locale` convenience form, which calls the same Scene Module and fails closed; accepting a validated plan is additive rather than a replacement. We deliberately reject a generic renderer tree/package, public recipe registry, render callbacks, author layout controls, and a universal sanitizer from V1: they would create another versioned Interface or move host trust policy into the wrong Module before a real need exists.
