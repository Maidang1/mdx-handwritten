# Share one Mark treatment vocabulary on mark and annotate

Authors kept reaching for “more marks” (for example circle-like emphasis) as if the language only had highlight, while `hw-mark` already owns a closed treatment set and `hw-annotate` only exposed a subset. We extend that closed set with `wavy` (doubt / unsettled) and `bracket` (side-clamped group, not a closed box), and make `hw-annotate`’s target `mark` accept the full Mark treatment vocabulary plus `none`. Mark treatment stays pure Visual style: without CSS it collapses to ordinary phrasing, does not inject decorative characters into the DOM, and never becomes a ninth Annotation gesture or a font choice. Strength remains mark-only; multi-line `bracket`/`circle` may paint imperfectly and docs steer authors to short targets. This prefers one shared author mental model over a quieter annotate-only subset, and prefers copy-safe degradation over no-CSS fake brackets.
