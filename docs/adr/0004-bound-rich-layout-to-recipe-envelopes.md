# Bound rich Annotation scene layout to recipe envelopes

An Annotation renderer may use spatial label rails only inside a declared Rich-layout envelope: V1 accepts `task-explainer` at `>=42rem` with at most two canonical-source lines and exactly six recipe roles, `mdtask-terminal` at `>=42rem` with exactly three command groups, seven task rows, and at most 59 ASCII columns, and `status-change` at `>=64rem` with exactly two panels and one factual verdict. Inside those envelopes, labels are collision-free only because source, label, and decorative connector occupy finite in-flow Grid cells; the core promises no global packing, obstacle routing, cross-wrapped-target connector, or browser measurement. Narrow, wrap-prone, over-capacity, fragmented-media, forced-colors, and unsupported-geometry scenes degrade as a whole to canonical source, numbered markers, and one complete legend; print uses that same linear form, no CSS retains canonical source plus the self-contained legend, and invalid semantics, references, or Scene plan V1 limits fail closed to canonical source alone. This deliberately makes the reader-first linear form the universal guarantee and keeps rich layout a deterministic recipe-owned enhancement, so authors never provide coordinates or repair collisions.
